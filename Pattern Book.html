  • The Lookup Pattern

    Goal: Find and return an element in a list. Return null if not found.
    [for each item in the list]
        if ([this is the item I want])
            return [item];
    return null





  • An example:
    private Account account(String type) {
        for (Account account : accounts)
            if (type.equals(accoutn.getType()))
                return account;
        return null;
    }

    NOTE: The name of a lookup function is the name of what is being returned.

  • Find All Matches

    Goal: Find all words in a list that contain the letter "z".
    Solution: Create a new list and add the matching words.
    private LinkedList[String] zWords(LinkedList[String] words) {
        LinkedList[String] matches = new LinkedList[String]();
        for (String word : words)
            if (word.contains("z"))
                matches.add(word);
        return matches;
    }





  • This creates a new list of Strings that stores the words that contain a 'z' in it in a LinkedList called 'matches'.
    It will continue to iterate through the LinkedList until the end and store words containing 'z' into the LinkedList called 'matches'.

  • Remove All Matches

    Goal: To remove all matches of words with a "z".

    Solution #1: Make a list of zWords, then remove them all at once.
        LinkedList[String] zWords = zWords(list);
        list.removeAll(zWords);

    Solution #2: Use an iterator.
        for (Iterator[String] it = list.iterator(); it.hasNext();)
            if (it.next().contains("z"))
                it.remove();





  • Solution #1 is simpler, but slower; it loops over the list twice. Solution #2 is more complex, but more efficient; it loops over the list once.

  • Remove One Match

    Goal: To remove only one match of words with a "z".

    Solution #1: Stop loop after removing to avoid an exception.
        for (String word : list)
            if (word.contains("z")) {
                list.remove(word);
                break;
    }

    Solution #2: Use an iterator.
        for (Iterator[String] it = list.iterator(); it.hasNext();)
            if (it.next().contains("z")) {
                it.remove();
                break;
    }





  • Stop looping when an item is removed.

            public void handle(){
                //do something;

Pattern Book 9

  • Immutable Property

    Goal: Create a property that never changes.

    public class SomeClass(){
        public final int value;
        public SomeClass(int value){
            this.value = value;
        }
        
        public final int getValue(){
            return value;
        }
    }





  • A property that never changes is a property that is never set.

    Never provide a set() method, so the property will never change.

    Final getter, no setter.

    Need to import 'javafx.beans.property.*;'

  • Read-Write Property

    Goal: Create a property readable, writeable and observable.

    public class SomeClass(){
        public IntegerProperty value = new SimpleIntegerProperty();
        public SomeClass(int value){
            this.value.set(value);
        }
        
        public final int getValue(){
            return value.get();
        }

        public final int setValue(int value){
            this.value.set(value);
        }

        public IntegerProperty valueProperty(){
            return value;
        }
     }





  • The valueProperty() method needs to be in that naming convention;
    it needes to be named xProperty(), where x is the name of the property.
    It is the accessor method that is needed to get the reference to the Property object itself.

    The text in red are very important.

  • Read-Only Property

    Goal: Create a property readable and observable, but NOT writable.

    public class SomeClass(){
        public IntegerProperty value = new SimpleIntegerProperty();
        public SomeClass(int value){
            this.value.set(value);
        }
        
        public final int getValue(){
            return value.get();
        }

        private final int setValue(int value){
            this.value.set(value);
        }

        public ReadOnlyIntegerProperty valueProperty(){
            return value;
        }
    }





  • Final getter and optional private setter.

    The property method returns a ReadOnlyProperty.
    This means that anything who tries to use this property will only see the get method and not the set method.

    It is much the same as the Read-Write Property pattern, only without the public setter method and that this returns a ReadOnlyProperty.

    The text in red are very important.

  • Immutable Property, Mutable State

    Goal: Create a property that is a reference to an object, where the property is immutable (property itself can't change), but the object may change its state.

    public class Customer(){
        private Account account;
        public Customer(){
            account = new Account("Mr Smith");
        }
        
        public Account final getAccount(){
            return account;
        }
        
    }





  • Not possible: customer.setAccount(new Account("Dr. Smith"));
    Still possible: customer.getAccount().setName("Dr. Smith");

    Final getter, no setter.

    A Customer has an Account, which is an object, and we want to make that Account a property
    of the Customer; the Customer has an Account property. We want to make that Account immutable with
    mutable state. What that means is, the Account reference cannot be changed; you can't replace
    that Account with another Account - it is always going to be that Account. But, the state
    inside that Account can change, so the balance inside that Account can change. It is just that
    you swap out that Accoutn for a completely different Account. It is an immutable property,
    something that can't change, but it has a mutable *state*, where the property's state *can* change.
    It looks like a regular immutable property in the method 'public final Account getAccount(){
    return account;}'. It's just that it contains an object, and that object, itself, may contain
    a mutable state.
